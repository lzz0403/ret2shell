事件 API 是 **回归终端** 提供给开发者的集成 API，开发者可以通过事件 API 来获取平台在比赛中产生的各种事件数据，然后根据这些数据来进行自己的数据分析和处理，也可以将它集成到一些自动化服务中，例如机器人播报等等。

在回归终端中，事件 API 是双向的，即可以通过事件 API 来获取平台产生的事件数据，也可以通过事件 API 来向平台发送事件数据。事件 API 通过正向 WebSocket 实现，开发者需要主动连接到回归终端的事件 API 端点并建立 WebSocket 连接，然后通过 WebSocket 协议来进行事件数据的传输。

## 连接

事件 API 的连接地址可以在比赛的管理页面复制，大概长这个样子：

```
wss://<domain.ret.sh.cn>/api/event/connect?game_id=<subscribed game>&token=<access token>
```

事件 API 是按照比赛区分的，一个订阅链接只会订阅一场比赛的相关事件。平台最多可以承接 128 个事件订阅连接，这个连接数被所有比赛共享，如果你同时举办 128 场比赛，并且每场比赛都接入事件服务，才会超出限额，因此在正常情况下是一定够用的，不需要关注这个限额。

> 但有一个可能超出限额的情况是，你的事件服务没有很好的管理连接，导致不断订阅同一场比赛的事件，旧连接又没有及时关闭，因此在超出限额时，你需要检查一下你的事件服务实现是否正确了。

平台采用正向 WebSocket 连接与主动推送，因此你的事件服务实现需要自行处理连接断开的情况，以及重连的逻辑。平台事件 API 没有持久化措施，这意味着在没有任何事件服务连接时，在这期间产生的所有事件都将会被直接丢弃。

## 事件数据格式

我们使用 JSON 作为统一传输格式，所有的事件会预先序列化为 JSON 字符串，然后通过 WebSocket 文本消息传递给所有连接的客户端。事件数据有一个通用的格式：

```json
{
  "<event_type>": {
    ...
  }
}
```

你可以通过判断 JSON 的根键来判断基础事件类型，然后在根据事件类型向下解析。所有的事件类型将在接下来的章节中详细介绍。
